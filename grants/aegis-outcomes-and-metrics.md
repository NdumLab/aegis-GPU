# Aegis Outcomes And Metrics

## Measurement Principle
Aegis should be evaluated as a troubleshooting-readiness product, not only as a content platform. The most important question is whether learners get better at identifying the correct layer, choosing the next safe action, and avoiding broad or unsafe remediation.

## Primary Outcomes

### 1. Improved Diagnostic Accuracy
Definition:
- learners more reliably identify the correct failure domain and next evidence source

Suggested measures:
- percent of scenarios with correct layer call
- percent of scenarios with correct next-step selection
- pre/post assessment score on incident classification questions

### 2. Safer Troubleshooting Behavior
Definition:
- learners reduce unsafe, over-broad, or evidence-destroying actions

Suggested measures:
- unsafe action rate
- false escalation rate
- rate of broad-fix selections before evidence collection

### 3. Faster Time To Correct Reasoning
Definition:
- learners reach the correct fault layer and corrective path more efficiently over time

Suggested measures:
- time to correct layer call
- number of wrong-path selections before correct containment or diagnosis
- step count to clean incident completion

### 4. Increased Completion And Readiness
Definition:
- learners finish more training paths cleanly and show stronger readiness signals

Suggested measures:
- clean incident finish rate
- lab completion rate
- quiz accuracy
- reasoning score improvement by category

## Recommended Core Metrics For A Pilot
Use a small core set first:
- correct layer call rate
- unsafe action rate
- clean incident finish rate
- quiz accuracy
- time to correct diagnosis

## Aegis-Aligned Metrics Already Supported By Current Product Direction
The current product already supports or points toward:
- reasoning scorecards
- reasoning progress summaries
- quiz scoring
- incident outcome tracking
- branch consequence outcomes
- pilot-ready JSON report export

## Pilot Evaluation Design

### Baseline
Before training:
- give learners a short scenario-based assessment
- measure initial quiz accuracy and layer-identification performance

### During Pilot
Track:
- labs attempted
- labs completed
- compromised vs clean finishes
- recurring reasoning weakness domains
- quiz performance

### Endline
After training:
- re-administer a scenario-based assessment
- compare before/after results on the same core metrics

## Sample Targets For A First Pilot
These are reasonable pilot targets, not guaranteed results:
- 20-30 point increase in scenario-based diagnostic accuracy
- 30% reduction in unsafe-action selections
- 25% improvement in clean incident finish rate
- 15-20 point increase in targeted quiz accuracy

## Evidence Outputs To Produce
At the end of a pilot, prepare:
- cohort summary report
- before/after metric table
- representative screenshots and browser-proof artifacts
- narrative analysis of common learner failure patterns
- recommendations for curriculum iteration

## Important Integrity Note
Aegis should not overclaim learning impact before a real pilot is complete. In grant applications, frame these as:
- target outcomes
- pilot evaluation metrics
- measurable success criteria

That is stronger and more credible than claiming proven impact prematurely.
